# 权限体系说明

## 概述

系统采用基于角色的权限控制（RBAC），通过 token 获取用户信息和权限。

## 权限设计

### 角色类型

- **管理者** (ADMIN) - 拥有所有权限
- **教练** (COACH) - 可查看和编辑学员、课程相关功能
- **餐饮后勤** (CHEF) - 可管理菜谱
- **清洁后勤** (CLEANER) - 可查看房间信息
- **前台** (RECEPTIONIST) - 可管理学员、查看营地和房间

### 权限标识

权限采用 `资源:操作` 的命名方式，例如：
- `dashboard:view` - 查看仪表板
- `student:create` - 创建学员
- `camp:delete` - 删除营地

## 使用方式

### 1. 在组件中使用权限检查

```typescript
import { usePermission } from '@/hooks/usePermission';
import { Permission } from '@/constants/permissions';

function MyComponent() {
  const { hasPermission, isAdmin } = usePermission();

  if (!hasPermission(Permission.STUDENT_CREATE)) {
    return <div>无权限</div>;
  }

  return <div>有权限的内容</div>;
}
```

### 2. 使用 PermissionGuard 组件

```typescript
import PermissionGuard from '@/components/PermissionGuard';
import { Permission } from '@/constants/permissions';

function MyPage() {
  return (
    <PermissionGuard permission={Permission.STUDENT_CREATE}>
      <div>需要创建学员权限才能看到的内容</div>
    </PermissionGuard>
  );
}
```

### 3. 检查多个权限

```typescript
// 需要任一权限
<PermissionGuard 
  permissions={[Permission.STUDENT_VIEW, Permission.STUDENT_EDIT]}
  requireAll={false}
>
  <div>有查看或编辑权限即可</div>
</PermissionGuard>

// 需要所有权限
<PermissionGuard 
  permissions={[Permission.STUDENT_VIEW, Permission.STUDENT_EDIT]}
  requireAll={true}
>
  <div>需要同时有查看和编辑权限</div>
</PermissionGuard>
```

### 4. 在 UserContext 中使用

```typescript
import { useUser } from '@/contexts/UserContext';
import { Permission } from '@/constants/permissions';

function MyComponent() {
  const { hasPermission, permissions, user } = useUser();

  // 检查单个权限
  const canCreate = hasPermission(Permission.STUDENT_CREATE);

  // 获取所有权限列表
  console.log(permissions);

  // 获取用户角色
  console.log(user?.role);
}
```

## Token 认证流程

1. 用户登录，获取 token
2. Token 存储在 localStorage（仅存储 token，不存储用户信息）
3. 每次请求自动在请求头携带 token
4. 通过 `/api/auth/me` 接口获取当前用户信息和权限
5. 根据用户角色计算权限列表

## 注意事项

1. **只存储 token**：本地只存储 token，用户信息通过接口获取
2. **权限实时计算**：权限根据用户角色实时计算，不存储在 token 中
3. **Token 失效处理**：如果 token 失效，自动清除并跳转登录页
4. **权限检查**：在需要权限控制的地方使用 `PermissionGuard` 或 `usePermission`
