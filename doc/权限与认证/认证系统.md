# 认证与权限系统说明

## 概述

系统采用基于 Token 的认证机制和基于角色的权限控制（RBAC）。

## 核心特性

### 1. Token 认证
- **只存储 token**：本地只存储 token，不存储用户信息
- **自动携带**：所有请求自动在请求头携带 token
- **通过接口获取**：用户信息和权限通过 `/api/auth/me` 接口获取

### 2. 权限体系
- **角色定义**：管理者、教练、餐饮后勤、清洁后勤、前台
- **权限标识**：采用 `资源:操作` 的命名方式
- **权限计算**：根据用户角色实时计算权限列表

## 认证流程

```
1. 用户登录
   ↓
2. 获取 token
   ↓
3. 存储 token 到 localStorage
   ↓
4. 通过 /api/auth/me 获取用户信息
   ↓
5. 根据用户角色计算权限
   ↓
6. 所有请求自动携带 token
```

## 接口说明

### 登录接口

#### 账号密码登录
```
POST /api/auth/login
Body: {
  username?: string,  // 账号（emp001）
  phone?: string,     // 手机号（13888880001）
  password: string    // 密码
}
Response: {
  success: true,
  data: {
    token: string,
    user: { ... }
  }
}
```

#### 手机号验证码登录
```
POST /api/auth/login-by-code
Body: {
  phone: string,  // 手机号
  code: string    // 验证码
}
Response: {
  success: true,
  data: {
    token: string,
    user: { ... }
  }
}
```

### 获取当前用户信息

```
GET /api/auth/me
Headers: {
  Authorization: Bearer {token}
}
Response: {
  success: true,
  data: {
    userId: string,
    id: string,
    name: string,
    phone: string,
    email: string,
    role: string,
    status: number,
    avatar: string
  }
}
```

## 权限使用示例

### 1. 使用 usePermission Hook

```typescript
import { usePermission } from '@/hooks/usePermission';
import { Permission } from '@/constants/permissions';

function MyComponent() {
  const { hasPermission, isAdmin, permissions } = usePermission();

  if (!hasPermission(Permission.STUDENT_CREATE)) {
    return <div>无权限</div>;
  }

  return <Button>创建学员</Button>;
}
```

### 2. 使用 PermissionGuard 组件

```typescript
import PermissionGuard from '@/components/PermissionGuard';
import { Permission } from '@/constants/permissions';

function StudentPage() {
  return (
    <PermissionGuard permission={Permission.STUDENT_VIEW}>
      <StudentList />
    </PermissionGuard>
  );
}
```

### 3. 在 UserContext 中使用

```typescript
import { useUser } from '@/contexts/UserContext';
import { Permission } from '@/constants/permissions';

function MyComponent() {
  const { hasPermission, user, permissions } = useUser();

  const canEdit = hasPermission(Permission.STUDENT_EDIT);
  
  return (
    <div>
      {canEdit && <Button>编辑</Button>}
    </div>
  );
}
```


## 安全注意事项

1. **Token 存储**：只存储在 localStorage，不存储敏感信息
2. **Token 过期**：实际项目中应实现 token 刷新机制
3. **权限验证**：前端权限检查仅用于 UI 控制，后端必须进行权限验证
4. **HTTPS**：生产环境必须使用 HTTPS 传输
