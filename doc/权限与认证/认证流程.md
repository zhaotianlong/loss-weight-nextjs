# 登录态处理流程说明

## 概述

系统实现了完善的登录态管理，包括 token 管理、自动跳转、路径保存等功能。

## 核心功能

### 1. Token 管理
- **存储位置**：localStorage（key: `auth_token`）
- **只存储 token**：不存储用户信息，用户信息通过接口获取
- **自动携带**：所有请求自动在请求头携带 token

### 2. 路径保存与恢复
- **保存位置**：sessionStorage（key: `redirect_path`）
- **保存时机**：
  - 未登录访问受保护页面时
  - Token 过期被拦截时
- **恢复时机**：登录成功后自动跳转到保存的路径

### 3. 自动跳转逻辑

#### 已登录用户访问登录页
- **场景**：用户已登录且 token 有效
- **行为**：自动跳转到之前保存的路径或首页（`/`）
- **实现**：登录页面检查 `isAuthenticated`，如果为 `true` 则跳转

#### 未登录用户访问受保护页面
- **场景**：用户未登录或 token 无效
- **行为**：
  1. 保存当前访问路径到 sessionStorage
  2. 跳转到登录页（`/login`）
- **实现**：`AuthGuard` 组件检测未登录状态

#### Token 过期
- **场景**：请求返回 401 错误
- **行为**：
  1. 清除 token
  2. 保存当前路径到 sessionStorage
  3. 显示错误提示
  4. 跳转到登录页
- **实现**：请求拦截器处理 401 错误

#### 退出登录
- **场景**：用户点击退出登录
- **行为**：
  1. 清除 token 和保存的路径
  2. 清除用户信息
  3. 跳转到登录页
- **实现**：`UserContext` 的 `logout` 方法

## 代码实现

### 1. Token 工具函数（`src/utils/auth.ts`）

```typescript
// 保存 token
setToken(token: string)

// 获取 token
getToken(): string | null

// 删除 token
removeToken()

// 保存登录前路径
saveRedirectPath(path: string)

// 获取并清除登录前路径
getAndClearRedirectPath(): string | null
```

### 2. 请求拦截器（`src/lib/request.ts`）

- **请求拦截器**：自动在请求头添加 token
- **响应拦截器**：处理 401 错误，自动清除 token 并跳转登录页

### 3. 登录页面（`app/login/page.tsx`）

- 检查用户是否已登录
- 如果已登录，跳转到保存的路径或首页
- 登录成功后，跳转到保存的路径或首页

### 4. 路由保护（`src/components/AuthGuard.tsx`）

- 检查用户登录状态
- 未登录时保存当前路径并跳转登录页
- 登录页面直接放行

### 5. 用户上下文（`src/contexts/UserContext.tsx`）

- 通过 token 获取用户信息
- 处理 token 失效情况
- 提供退出登录功能

## 使用场景示例

### 场景 1：正常登录流程
1. 用户访问 `/student`（未登录）
2. `AuthGuard` 检测到未登录，保存路径 `/student`，跳转到 `/login`
3. 用户输入账号密码登录
4. 登录成功，跳转到 `/student`（恢复之前保存的路径）

### 场景 2：已登录用户访问登录页
1. 用户已登录，直接访问 `/login`
2. 登录页面检测到已登录，自动跳转到首页 `/`

### 场景 3：Token 过期
1. 用户正在浏览 `/camp-manage` 页面
2. Token 过期，发送请求返回 401
3. 请求拦截器检测到 401，保存路径 `/camp-manage`，清除 token，跳转到 `/login`
4. 用户重新登录，登录成功后跳转到 `/camp-manage`

### 场景 4：退出登录
1. 用户在任意页面点击退出登录
2. `logout` 方法清除 token 和用户信息
3. 跳转到登录页 `/login`

## 注意事项

1. **路径保存**：使用 `sessionStorage` 保存路径，浏览器关闭后清除
2. **防止重复跳转**：使用 `isRedirectingToLogin` 标志防止重复跳转
3. **登录页面检查**：请求拦截器中检查当前路径，避免在登录页重复跳转
4. **路由替换**：登录成功跳转使用 `router.replace` 而不是 `router.push`，避免浏览器历史记录中有登录页
5. **加载状态**：登录页面在检查登录状态时显示加载状态，避免闪烁

## 安全考虑

1. **Token 存储**：Token 存储在 localStorage，需要注意 XSS 攻击风险
2. **Token 过期**：后端应正确验证 token 有效期
3. **路径验证**：登录后跳转的路径应该验证有效性，避免开放重定向漏洞
4. **HTTPS**：生产环境必须使用 HTTPS 传输
