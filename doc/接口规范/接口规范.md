# 减肥训练营管理系统 - API 规范

## 1. API 基础信息

### 基础 URL
```
http://localhost:3000/api
```

### 请求格式
```
Content-Type: application/json
Authorization: Bearer {JWT_TOKEN}
```

### 响应格式
所有响应均为 JSON 格式，遵循以下结构：

#### 成功响应（2xx）
```json
{
  "success": true,
  "data": {
    // 具体数据
  },
  "message": "操作成功"
}
```

#### 错误响应（4xx/5xx）
```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "错误描述"
  }
}
```

## 2. 认证接口

### 登录
```
POST /auth/login
```

**请求体**
```json
{
  "email": "user@example.com",
  "password": "password123"
}
```

**响应**
```json
{
  "success": true,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "id": "user_1",
      "name": "张三",
      "email": "user@example.com",
      "role": "admin"
    }
  }
}
```

### 登出
```
POST /auth/logout
```

### 刷新令牌
```
POST /auth/refresh
```

## 10. 错误代码

| 代码 | 含义 | HTTP 状态 |
|------|------|---------|
| UNAUTHORIZED | 未授权/令牌过期 | 401 |
| FORBIDDEN | 无权限 | 403 |
| NOT_FOUND | 资源不存在 | 404 |
| CONFLICT | 资源冲突 | 409 |
| VALIDATION_ERROR | 数据验证失败 | 400 |
| PAYMENT_FAILED | 支付失败 | 402 |
| SERVER_ERROR | 服务器错误 | 500 |

## 11. 分页规则

所有列表接口都支持分页：

```
?page=1&limit=20&sort=createdAt&order=desc
```

响应会包含分页信息：

```json
{
  "success": true,
  "data": [...],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 150,
    "totalPages": 8
  }
}
```

## 12. 速率限制

- 普通用户：100 请求/分钟
- VIP 用户：500 请求/分钟
- 管理员：无限制

限制信息在响应头中：

```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 87
X-RateLimit-Reset: 1672531200
```
