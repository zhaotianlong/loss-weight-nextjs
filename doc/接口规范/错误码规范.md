# 错误码规范

## 概述

错误码分为两类：**网络错误**和**业务错误**

## 1. 网络错误

使用传统 HTTP 状态码（4xx、5xx）

### 常见网络错误码

- **400**: 请求参数错误（格式错误、必填字段缺失等）
- **401**: 未授权（Token 无效或过期）
- **403**: 拒绝访问（无权限）
- **404**: 资源不存在
- **500**: 服务器内部错误

### 响应格式

```json
{
  "success": false,
  "message": "错误描述"
}
```

## 2. 业务错误

HTTP 响应统一为 **200**，但在响应数据中包含 `code` 和 `msg` 字段

### 响应格式

```json
{
  "code": 4001,  // 业务错误码，200 表示成功，其他值表示业务错误
  "msg": "业务错误描述",
  "success": false,
  "data": null
}
```

### 业务错误码规范

- **200**: 成功
- **4001-4999**: 业务逻辑错误
  - 4001: 房间有学员无法删除
  - 其他业务错误码待定义

## 3. 成功响应格式

```json
{
  "code": 200,
  "success": true,
  "msg": "操作成功",
  "data": { ... },
  "meta": { ... }  // 可选，用于分页
}
```

## 前端处理逻辑

1. 响应拦截器先检查 `code` 字段
   - 如果 `code` 存在且不为 200，则显示 `msg` 作为业务错误
   - HTTP 状态码为 200 的情况下，`code` 不为 200 表示业务错误

2. 错误拦截器处理 HTTP 状态码错误（4xx、5xx）
   - 这些是网络层面的错误
   - 显示相应的错误消息
