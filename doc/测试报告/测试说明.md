# 单元测试文档

## 测试框架配置

项目使用以下测试框架：
- **Jest**: 测试运行器和断言库
- **React Testing Library**: React 组件测试
- **@testing-library/user-event**: 用户交互模拟

## 安装依赖

```bash
npm install
```

## 运行测试

```bash
# 运行所有测试
npm test

# 监听模式运行测试
npm run test:watch

# 生成覆盖率报告
npm run test:coverage
```

## 测试最佳实践

### 1. 测试命名
- 使用描述性的测试名称
- 遵循 "应该..." 的命名模式（中文项目）

### 2. 测试结构
```typescript
describe('组件/函数名', () => {
  beforeEach(() => {
    // 设置测试环境
  })

  it('应该执行某个行为', () => {
    // 测试代码
  })
})
```

### 3. Mock策略
- 使用 `jest.mock()` 模拟外部依赖
- Mock Next.js router、localStorage、sessionStorage
- Mock API请求（使用MSW或直接mock）

### 4. 断言
- 使用 `@testing-library/jest-dom` 的匹配器
- 优先测试用户可见的行为
- 避免测试实现细节

### 5. 覆盖率目标
- 工具函数: 100%
- Service层: 80%+
- 组件: 70%+
- 页面: 60%+

## 注意事项

1. **Mock数据**: 测试应该使用独立的mock数据，不依赖真实数据
2. **异步测试**: 使用 `async/await` 和 `waitFor` 处理异步操作
3. **清理**: 在 `beforeEach` 或 `afterEach` 中清理状态
4. **隔离**: 每个测试应该独立运行，不依赖其他测试的状态
