## 已知问题

### 1. React 19 兼容性
**问题**: React 19 与 @testing-library/react 14 不完全兼容

**影响**: 页面组件测试可能失败

**临时解决方案**: 
- 使用 `--legacy-peer-deps` 安装依赖
- 简化页面测试，专注于逻辑而非渲染

### 2. 密码测试Mock
**问题**: crypto API的mock需要更精确的设置

**状态**: 已修复，使用真实crypto API进行测试

### 3. TextEncoder Polyfill
**问题**: Node.js环境缺少TextEncoder

**状态**: 已在jest.setup.js中添加polyfill

## 测试最佳实践

### 1. 测试结构
```typescript
describe('功能模块', () => {
  beforeEach(() => {
    // 设置
  })

  it('应该执行某个行为', () => {
    // 测试
  })
})
```

### 2. Mock策略
- 使用 `jest.mock()` 模拟外部依赖
- Mock Next.js router、localStorage等
- 使用MSW或直接mock API请求

### 3. 断言
- 使用 `@testing-library/jest-dom` 匹配器
- 测试用户可见的行为
- 避免测试实现细节
