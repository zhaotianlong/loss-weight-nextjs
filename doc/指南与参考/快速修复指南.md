### 2. 优化 MSW 初始化

**文件：** `src/mocks/InitMock.tsx`

**当前代码：**
```typescript
if (!started) return null; // 阻塞渲染
```

**修复为：**
```typescript
export default function InitMock() {
  useEffect(() => {
    if (process.env.NODE_ENV !== 'development') return;

    (async () => {
      try {
        const { worker } = await import('./browser');
        await worker.start({ onUnhandledRequest: 'bypass' });
        console.info('[MSW] Mock worker started');
      } catch (err) {
        console.error('[MSW] Failed to start worker', err);
        // 不阻塞应用运行
      }
    })();
  }, []);

  // 不阻塞渲染，始终返回 null
  return null;
}
```

### 3. 示例：将页面改为服务端组件

**文件：** `app/student/page.tsx`

**当前代码（客户端组件）：**
```typescript

export default function StudentPage() {
  const [data, setData] = useState([]);
  
  useEffect(() => {
    const fetchData = async () => {
      const res = await getStudentList();
      setData(res.data);

    fetchData();
  }, []);

  return <ProTable dataSource={data} />;
}
```

**修复为（服务端组件 + 客户端交互组件）：**


**步骤 2：页面改为服务端组件**
```typescript
// app/student/page.tsx
// 移除 'use client'

import StudentTable from './components/StudentTable';
import { getStudentList } from '@/service/student';

export default async function StudentPage() {
  // 服务端获取数据
```

**注意：** 如果 `getStudentList` 使用 Axios，需要：
1. 创建服务端版本的请求函数（使用 `fetch` 而不是 Axios）
2. 或使用 Next.js Route Handlers 作为中间层

## ⚠️ 注意事项

1. **服务端组件限制：**
   - 不能使用 `useState`、`useEffect` 等 hooks
   - 不能使用浏览器 API（`window`、`localStorage` 等）
   - 不能使用事件处理器

2. **数据获取：**
   - 服务端组件可以直接 `await` 异步函数
   - 如果使用 Axios，需要创建服务端版本（使用 `fetch`）

3. **渐进式迁移：**
   - 不需要一次性修改所有页面
   - 可以逐个页面迁移
   - 优先迁移数据展示页面
